[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the workflow architecture. The entire system is governed by a deterministic ADK Graph Workflow.
When an employee submits an expense, it first enters our Security Checkpoint node. This node handles immediate input verification, PII scrubbing, and prompt injection detection.
If a security risk is found, the system routes the request to a Security Block node and terminates.
Otherwise, the claim goes to our main Orchestrator Agent.
The Orchestrator coordinates two specialized LlmAgents via AgentTools: the Policy Auditor and the Anomaly Detector.
Both of these specialists use a local Model Context Protocol server, or MCP server. The Policy Auditor queries the MCP tools to look up category-specific spending guidelines and employee claim histories. Meanwhile, the Anomaly Detector queries the MCP budget tool to verify remaining department budgets.
The specialists return their findings to the Orchestrator, which decides to Approve, Reject, or route the claim to a Review Router.
If the Orchestrator flags a suspicious claim, the Review Router sends it to a Human Review node where the execution pauses and awaits a manager's manual input. Otherwise, it proceeds to the Final Output node.
